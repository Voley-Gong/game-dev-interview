## 参考答案

### ✅ 核心要点

1. **背包问题本质：有限容量下的价值最大化决策**：背包容量 W，n 件物品各有重量 wᵢ 和价值 vᵢ，目标是在不超重的前提下选物品使总价值最大。这是 NP-Hard 的组合优化问题，但用动态规划可在 `O(nW)` 伪多项式时间内求精确解。游戏中用于自动装填推荐、出门装备最优选择、资源采集规划、商店购物预算。
2. **三种核心变体：0-1 背包、完全背包、多重背包**：0-1 背包每件物品最多选一次（出门装备）；完全背包每件物品可选无限次（金币商店买药水）；多重背包每件物品有数量上限 cᵢ（库存有限的合成材料）。三者状态定义相同（`dp[i][w]`），但转移方程和遍历方向不同，混淆会得到错误解。
3. **DP 核心三要素：状态定义、状态转移、遍历顺序**：状态 `dp[i][w]` 表示"前 i 件物品、容量 w 时的最大价值"；转移 `dp[i][w] = max(dp[i-1][w], dp[i-1][w-wᵢ] + vᵢ)`（不选/选第 i 件）；遍历顺序决定是 0-1（外层物品、内层容量逆序）还是完全（内层容量正序）。把这三点讲清楚，背包题就通了。
4. **空间优化：二维 DP → 滚动数组 → 一维数组**：朴素二维 `dp[n][W]` 占 `O(nW)` 空间，n=1000、W=10000 时就是 40MB。观察到 `dp[i]` 只依赖 `dp[i-1]`，用两行滚动数组降到 `O(2W)`；进一步发现 0-1 背包内层逆序遍历时一维数组即可，空间 `O(W)`。这是面试必考的优化点。
5. **游戏实战场景：不是每题都上 DP**：背包容量小（≤20 格）物品少（≤50）时 DP 秒解；物品上千时 `O(nW)` 太慢，改用贪心（按性价比 v/w 排序）求近似解，或分支限界+剪枝求精确解。还要注意游戏背包常带额外约束（同类限数、套装加成、负重影响移速），纯背包 DP 只是基础模型。
6. **路径回溯：DP 只给最优值，还要还原"选了哪些物品"**：面试常追问"选了哪几件"。需要在求 `dp` 的同时记录选择决策（`choice[i][w]` 标记是否选了第 i 件），最后从 `dp[n][W]` 反向回溯。游戏的"自动装填推荐"功能必须回溯路径，否则只能告诉玩家"最大价值 150"却不能列出推荐装备清单。

### 📖 深度展开

**1. 0-1 背包：从二维 DP 到一维滚动数组优化**

```typescript
interface Item { name: string; weight: number; value: number; }

// === 方案 A：朴素二维 DP + 路径回溯（清晰但费空间）===
function knapsack01(items: Item[], capacity: number): { maxValue: number; chosen: Item[] } {
  const n = items.length;
  // dp[i][w] = 前 i 件物品、容量 w 时的最大价值
  const dp: number[][] = Array.from({ length: n + 1 }, () => new Array(capacity + 1).fill(0));
  const choice: boolean[][] = Array.from({ length: n + 1 }, () => new Array(capacity + 1).fill(false));

  for (let i = 1; i <= n; i++) {
    const { weight, value } = items[i - 1];
    for (let w = 0; w <= capacity; w++) {
      dp[i][w] = dp[i - 1][w];            // 默认不选第 i 件
      if (w >= weight && dp[i - 1][w - weight] + value > dp[i][w]) {
        dp[i][w] = dp[i - 1][w - weight] + value;  // 选第 i 件更优
        choice[i][w] = true;
      }
    }
  }

  // 路径回溯：从终点反向推导选了哪些物品
  const chosen: Item[] = [];
  let w = capacity;
  for (let i = n; i >= 1; i--) {
    if (choice[i][w]) { chosen.push(items[i - 1]); w -= items[i - 1].weight; }
  }
  return { maxValue: dp[n][capacity], chosen };
}

// === 方案 B：一维滚动数组优化（空间 O(W)，但丢失路径）===
function knapsack01Optimized(items: Item[], capacity: number): number {
  const dp = new Array(capacity + 1).fill(0);  // 只用一维
  for (const { weight, value } of items) {
    // ⚠️ 关键：容量必须逆序遍历！否则同一物品会被重复选取（变成完全背包）
    for (let w = capacity; w >= weight; w--) {
      dp[w] = Math.max(dp[w], dp[w - weight] + value);
    }
  }
  return dp[capacity];
}
// 出门装备推荐：负重 20，选剑(7,30)、盾(5,20)、弓(4,15)、药(3,8)、书(6,25)
// knapsack01(equipment, 20) → 最大价值 98，推荐：剑+盾+弓+药
```

**2. 三种背包变体：转移方程与遍历方向的差异**

| 变体 | 物品数量 | 转移方程 | 内层容量遍历 | 时间复杂度 | 游戏场景 |
|------|---------|---------|------------|----------|---------|
| **0-1 背包** | 每件 0/1 | `dp[w]=max(dp[w], dp[w-wᵢ]+vᵢ)` | **逆序** ↓ | O(nW) | 出门装备、唯一任务奖励 |
| **完全背包** | 无限件 | `dp[w]=max(dp[w], dp[w-wᵢ]+vᵢ)` | **正序** ↑ | O(nW) | 商店买药水、金币换资源 |
| **多重背包** | 有限 cᵢ 件 | 二进制拆分转 0-1 | 逆序 | O(n·logc·W) | 限量合成材料、库存物品 |

```typescript
// 完全背包：物品无限，内层正序遍历（允许同一物品被多次选取）
function knapsackComplete(items: Item[], capacity: number): number {
  const dp = new Array(capacity + 1).fill(0);
  for (const { weight, value } of items) {
    for (let w = weight; w <= capacity; w++) {  // ⚠️ 正序：w 从小到大
      dp[w] = Math.max(dp[w], dp[w - weight] + value);
    }
  }
  return dp[capacity];
}
// 商店：1 金币容量，药水(花费2,回复30HP)无限供应，求 10 金币能买多少 HP
// → 完全背包，dp[10] = 150（买 5 瓶药水）

// 多重背包：二进制拆分——把 c 件拆成 1,2,4,...,余数，转成 0-1 背包
function knapsackMultiple(items: (Item & { count: number })[], capacity: number): number {
  const dp = new Array(capacity + 1).fill(0);
  for (const { weight, value, count } of items) {
    // 把 count 件拆成 log(count) 个"打包件"，等价但更快
    let k = 1, remain = count;
    while (remain > 0) {
      const take = Math.min(k, remain);
      const w = take * weight, v = take * value;
      for (let c = capacity; c >= w; c--) dp[c] = Math.max(dp[c], dp[c - w] + v);
      remain -= take; k *= 2;
    }
  }
  return dp[capacity];
}
```

          容量→  0   100  150  250  300  350  450  500
  无物品        0    0    0    0    0    0    0    0

| 求解方法 | 结果 | 是否最优 | 耗时 | 适用场景 |
|---------|------|---------|------|---------|
| 贪心（性价比排序） | 48 | ❌ 近似 | <1ms | 物品多、要求快 |
| **0-1 背包 DP** | **65** | ✅ 精确 | ~2ms | 物品 ≤50、需精确 |
| 暴力枚举 2ⁿ | 65 | ✅ | ~数秒 | n ≤ 20 验证用 |
| 分支限界 + 剪枝 | 65 | ✅ | ~1ms | n 大但容量约束紧 |

### ⚡ 实战经验

- **贪心不是背包的最优解**：新手用"按性价比 v/w 排序贪心"做自动装填，结果推荐的装备组合实际战力比 DP 解低 30%。背包是离散决策，贪心只在分数背包（物品可分割）下最优，游戏背包是 0-1 离散的必须用 DP。上线前用小规模暴力枚举对拍验证 DP 正确性。
- **物品上千时 DP 太慢**：某 RPG 有 2000 件装备、负重上限 100，`O(nW)=200000` 还好；但负重按 0.1 精度算时 W=1000，`O=2×10⁶`，移动端单次推荐卡 80ms。解法：把重量归一化到整数格（0.1→1），或用分支限界+上界估计剪枝，实测 95% 场景剪枝后 <5ms。
- **路径回溯漏选是常见 Bug**：只存 `maxValue` 没存 `choice` 数组，自动推荐只能显示"最大战力 65"却列不出装备清单，玩家无法使用。必须同时维护决策记录并回溯。某项目上线后才发现这个缺陷，紧急补丁加了 `chosen` 数组。
- **套装加成破坏纯背包模型**：装备 2 件"火焰套装"额外 +20 战力，这是状态依赖的奖励，标准背包 DP 无法表达。需要把"套装件数"加进状态：`dp[i][w][setCount]`，状态空间乘以套装件数上限，复杂度上升。或用迭代：先按无加成求 DP，再贪心补齐套装，近似但快。
- **负重影响移速要建模成多目标优化**：游戏里负重超阈值移速下降，纯价值最大化 DP 会塞满背包导致移速暴跌、实际体验更差。把"移速惩罚"折算成价值负权重：`effectiveValue = value - overweightPenalty`，或用帕累托前沿保留多组解让玩家选。这是背包问题游戏化的常见扩展。
