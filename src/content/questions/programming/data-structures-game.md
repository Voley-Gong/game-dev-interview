## 参考答案

### ✅ 核心要点

1. **场景管理**：四叉树（2D）/ 八叉树（3D）加速空间查询
2. **寻路**：A* 算法 + 网格/导航网格
3. **状态管理**：有限状态机（FSM）管理角色/敌人 AI
4. **对象池**：预分配 + 复用，避免 GC
5. **消息队列**：事件排序和处理

### 📖 深度展开

**1. 四叉树 / 八叉树（空间划分）**

```
场景中 10000 个物体，需要找到玩家附近的敌人：
- 暴力遍历：O(n) = O(10000)
- 四叉树查询：O(log n) ≈ O(13)

原理：递归将空间四等分，只在相关子空间中查询
用途：碰撞检测、视野裁剪、范围技能
```

**2. A* 寻路**

```typescript
// 核心公式：f(n) = g(n) + h(n)
// g(n) = 起点到 n 的实际代价
// h(n) = n 到终点的启发式估算
// 使用优先队列（最小堆）取 f 最小的节点

// 优化技巧：
// - 使用导航网格替代格子（更自然）
// - 分帧寻路（大地图不一次算完）
// - 路径平滑（去掉不必要的拐点）
// - 多线程寻路
```

**3. 有限状态机（FSM）**

```
角色 AI 状态转换：
  Idle → Patrol → Chase → Attack → Flee → Dead

每个状态定义：
  - Enter：进入时初始化
  - Update：持续执行
  - Exit：离开时清理
  - Transition：状态转换条件
```

**4. 对象池**

```typescript
class ObjectPool<T> {
  private pool: T[] = [];
  
  get(factory: () => T): T {
    return this.pool.pop() || factory();
  }
  
  put(obj: T, reset: (obj: T) => void): void {
    reset(obj);
    this.pool.push(obj);
  }
}
```

### ⚡ 实战经验

- **A* 不是万能的**：RTS 大量单位寻路用 Flow Field 更合适
- **状态机可以可视化**：编辑器中拖拽配置状态机，策划也能用
- **对象池大小**：根据实际峰值用量设置，太浪费内存、太小无效果
- **行为树 vs FSM**：复杂 AI 用行为树，简单 AI 用 FSM 就够了
