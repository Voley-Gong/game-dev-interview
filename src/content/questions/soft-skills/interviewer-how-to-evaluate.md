## 参考答案

### ✅ 核心要点

1. **先定义人才画像**：面试前明确"这个岗位需要什么样的人"——技术深度、广度、协作能力、成长潜力的权重各是多少
2. **分层考察**：基础知识（过滤）→ 工程能力（核心）→ 系统思维（区分）→ 文化匹配（决定）
3. **用结构化面试减少偏见**：同一岗位所有候选人答相同的题，用统一评分表打分
4. **信号词敏锐度**：候选人在回答中透露的习惯和态度比答案本身更重要
5. **sell 候选人**：面试是双向的，优秀的人选也在评估你

### 📖 深度展开

**面试环节设计（游戏客户端开发岗位为例）：**

```
┌─────────────────────────────────────────────────────────┐
│  环节          │ 时长   │ 考察重点          │ 面试官     │
├─────────────────────────────────────────────────────────┤
│  笔试/编程题   │ 60min  │ 基础编码能力      │ HR/自动   │
│  技术一面      │ 60min  │ 基础+工程实践     │ 中级开发  │
│  技术二面      │ 60min  │ 架构+性能优化     │ 主程/资深 │
│  项目深挖面    │ 45min  │ 项目经验真实性    │ 主程      │
│  综合面        │ 30min  │ 协作/文化/潜力    │ 制作人/总监│
└─────────────────────────────────────────────────────────┘

关键原则：
- 每轮有明确目标，不重复考察
- 前一轮的反馈在下一轮前同步
- 淘汰决策当天做出，不要拖
```

**四层考察模型：**

| 层级 | 考察内容 | 题目示例 | 判断标准 |
|------|----------|----------|----------|
| L1 基础过滤 | 编程基础、CS 基础 | "手写一个对象池" | 能写出可运行代码 |
| L2 工程能力 | 真实开发场景 | "游戏运行时内存持续增长，如何排查？" | 能说出工具链和排查思路 |
| L3 系统思维 | 架构设计、权衡 | "设计一个热更新系统，考虑哪些维度？" | 能做多方案对比和 tradeoff 分析 |
| L4 潜力判断 | 学习能力、协作 | "如果让你做一个没接触过的系统，你会怎么开始？" | 展现方法论和成长意愿 |

**深挖项目经验的高效提问法：**

```
原则：STAR 反向应用——让候选人讲完整故事，你负责挖细节验真

层层递进的问题设计：

第1层（开放）："说说你在XX项目中最有成就感的技术工作？"

第2层（聚焦）："你说你优化了渲染性能，具体从多少提升到多少？"

第3层（验证）："DrawCall 从 500 降到 100，你用了哪些具体手段？"

第4层（边界）："如果继续优化，瓶颈会转移到哪里？你当时为什么停在 100？"

第5层（反思）："现在回头看，有没有更好的方案？"

信号判断：
✅ 好信号：能说清 why、能自我反思、量化指标准确
❌ 坏信号：只说 what 不说 why、数字模糊、全靠"我们团队"
```

**文化匹配度评估（游戏行业特有）：**

```
关键问题及解读：

1. "你平时玩什么游戏？最近在玩什么？"
   - 不是看玩得多不多，而是看他是否对游戏有热情和理解
   - 能说出具体喜欢的设计点 → 有产品思维
   - "不太玩游戏，只做技术" → 在游戏行业会受限

2. "你跟策划意见不一致的时候怎么处理？"
   - "用代码实现他说的就行了" → 太被动
   - "吵到制作人那里" → 太对抗
   - "先理解他的设计意图，再用技术角度提供多个方案" → 成熟

3. "项目上线后看到玩家骂你写的功能，什么感受？"
   - 感到难受但能理性分析 → 有责任心
   - "策划设计的，跟我没关系" → 缺乏主人翁意识

4. "你理想中的团队是什么样的？"
   - 描述协作方式、学习氛围 → 有团队意识
   - 只关注技术栈、薪资 → 动机偏差
```

### ⚡ 实战经验

- **警惕「答题型」候选人**：算法题刷得很溜，但追问项目细节就卡壳。游戏开发要的是工程能力，不是竞赛选手
- **面试题要持续迭代**：同一个题用半年，网上就会有"标准答案"。每 3-6 个月换一批，或者基于内部项目出题
- **注意 Halo Effect**：候选人某一方面特别强（比如名校/大厂背景），不要因此整体打高分。逐项独立评分
- **招人比培养人重要**：面试时发现文化不合的信号，不要因为"缺人"就降低标准。错招一个人的代价远大于空位等人的代价
- **给候选人好体验**：面试结束给建设性反馈、及时给结论。优秀的候选人手里有多个 Offer，面试体验直接影响接受率
- **作为候选人面试公司**：反过来也要评估对方——团队技术水平、项目前景、管理风格。好的面试官会欢迎你的提问
