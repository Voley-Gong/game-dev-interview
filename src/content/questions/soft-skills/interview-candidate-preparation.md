## 参考答案

### ✅ 核心要点

1. **分层准备**：技术深度 → 项目经验 → 行为问题 → 反问环节，四层递进
2. **建立「故事库」**：用 STAR 法则准备 6-8 个核心故事，覆盖不同能力维度
3. **研究目标公司**：玩他们的游戏、看技术博客/分享、了解团队架构和技术栈
4. **模拟面试**：至少做 2-3 次模拟，重点练「说人话」而非背稿子
5. **准备好反问**：好问题比好答案更能展现思考深度

### 📖 深度展开

**面试准备时间线（建议提前 3-4 周）：**

```
Week 1: 基础梳理
├── 梳理项目经历，列出 3-5 个核心项目
├── 为每个项目准备：背景、你的角色、技术难点、成果数据
├── 刷基础题（数据结构、算法、游戏引擎原理）
└── 整理技术博客/作品集链接

Week 2: 深度准备
├── 研究目标公司的产品、技术栈、近期动态
├── 针对岗位 JD 逐条对照自己的经验，找出 gap
├── 准备故事库（STAR 格式，见下方）
├── 系统设计题练习（如果是高级岗位）
└── 准备一份「我想了解的问题」清单

Week 3: 模拟实战
├── 找朋友/同事做模拟面试
├── 录音回放，检查语速、逻辑、冗余
├── 针对薄弱点做专项突破
└── 准备 2-3 个深度的技术话题，随时可展开

Week 4: 状态调整
├── 不再大量背新内容，而是复习巩固
├── 调整作息，确保面试当天精神好
├── 准备面试当天的后勤（路线、着装、设备）
└── 心态建设：把它当成一次技术交流，而非审讯
```

**故事库模板（6-8 个故事覆盖核心维度）：**

| 能力维度 | 故事主题 | STAR 结构要点 |
|----------|----------|---------------|
| 技术攻坚 | 性能优化/架构重构 | 问题有多严重 → 你怎么分析 → 怎么解决 → 结果数据 |
| 团队管理 | 带新人/处理低绩效 | 挑战是什么 → 你的管理动作 → 成员变化 → 团队效果 |
| 跨部门协作 | 推动一个跨团队项目 | 分歧在哪 → 你怎么协调 → 最终结果 |
| 失败复盘 | 一个搞砸的项目 | 诚实描述失败 → 根因分析 → 学到了什么 → 后续改进 |
| 技术决策 | 技术选型/方案评审 | 选项有哪些 → 评估标准 → 最终决策 → 后续验证 |
| 主动性 | 你主动推动的一个改进 | 发现什么问题 → 你做了什么 → 对团队的影响 |

**反问环节的高质量问题模板：**

```
关于岗位/团队：
  - 这个岗位过去半年最大的挑战是什么？
  - 团队的技术债务状况如何？有没有专项时间？
  - 团队成员的成长路径是怎么设计的？

关于技术/产品：
  - 你们在 [公司游戏名] 中遇到的最大技术挑战是什么？
  - 从 [某版本] 到现在，架构上做了哪些重大演进？
  - 你们怎么看 [新技术趋势] 在项目中的应用前景？

关于协作/文化：
  - 技术和策划之间出现分歧时，通常怎么解决？
  - 团队的决策机制是怎样的？是技术负责人拍板还是集体讨论？
  - 你（面试官）在这个团队中最享受的部分是什么？
```

### ⚡ 实战经验

- **研究公司产品是基本功课**：面试官最反感的不是技术不行，而是「你都没玩过我们的游戏就来了」。至少玩 10 小时目标公司的核心产品
- **薪资谈判不要在技术面聊**：技术面只谈技术和匹配度。薪资留给 HR 面，手握 offer 后再谈
- **面试是双向信息收集**：注意观察面试官的状态、团队氛围、办公环境，这些都在告诉你这家公司适不适合
- **面试后 24 小时内发感谢信**：简短即可，「感谢今天的时间，聊到 XX 话题很有收获」。不是必须但可以加分
- **不要贬低前公司/前同事**：面试官会想「他以后离开我们是不是也这样说」。讲不愉快的经历时聚焦「学到了什么」
