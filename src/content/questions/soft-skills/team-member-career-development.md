## 参考答案

### ✅ 核心要点

1. **双向画像**：了解成员的个人目标 + 评估其能力矩阵 = 拼出发展路径
2. **IDP（Individual Development Plan）**：每半年共创一份个人发展计划，3-5 个具体目标
3. **成长 ladder 可视化**：让每个人知道"下一步是什么""达到什么标准可以晋升"
4. **给机会而非只给指导**：挑战性任务是最有效的成长催化剂，管理者要做的是"敢放手"
5. **定期校准**：季度 1on1 跟踪 IDP 进展，及时调整方向

### 📖 深度展开

**游戏客户端团队的典型成长路径：**

```

  │
  ├─▶ 中级工程师 (Mid)
  │      核心：能负责子系统，参与方案设计
  │      周期：2-4年

  │      │                      │
  │      │      ┌───────────────┘
  │      │      ▼               ▼
```

**IDP（个人发展计划）模板：**

```markdown
## 2026 H2 个人发展计划 - [姓名]

### 职业目标（本人填写）
- 长期方向：技术专家 / 技术管理 / 转型PM（选一）
- 1年目标：能独立负责战斗系统的架构设计
- 成功时的样子：在团队内做一次战斗系统架构的技术分享

### 能力矩阵评估（管理者填写，与本人讨论）
| 维度 | 当前 | 目标 | Gap |
|------|------|------|-----|
| 技术深度 | L3 能做方案 | L4 能做架构 | 需学习设计模式+架构方法论 |
| 项目把控 | L2 能拆任务 | L3 能做里程碑规划 | 需参与1次完整项目规划 |
| 沟通协作 | L3 能跨组沟通 | L3 ✅ | 达标 |
| 技术影响力 | L2 组内分享 | L3 全公司分享 | 需准备1次高质量分享 |

### 本周期具体行动（共创）
1. [技术] 负责战斗系统重构模块，每周做一次方案评审（提升架构能力）
2. [项目] 参与下个项目的技术排期过程，学习里程碑规划
3. [影响力] Q3 做一次全公司级别的"战斗系统性能优化"技术分享
4. [学习] 读完《游戏编程模式》，输出读书笔记到团队Wiki

### 支持承诺（管理者承诺）
- 每两周 1on1，重点关注战斗系统重构的方案设计
- 介绍公司架构师做他的 mentor
- 下次技术分享会优先安排他的议题
```

**1on1 中引导职业规划的话术：**

```
开场（轻松）：
"最近工作感觉怎么样？有没有什么特别有成就感或者特别卡的地方？"

引导（探索）：
"你觉得自己在团队里最被认可的能力是什么？"
"你理想中，一年后的自己在做什么类型的工作？"
"有没有什么技术方向是你特别想深入，但平时没机会接触的？"

现实检验（温和但诚实）：
"从我的观察来看，你的 XX 能力已经很扎实了。
YY 方面如果能再有 2-3 个实战项目的积累，
就具备晋升评审的条件了。你怎么看？"

承诺支持：
"下个项目我安排你负责 XX 模块的架构，
这对你目标的达成会很有帮助。
我每两周和你 sync 一次进展，随时帮你扫清障碍。"
```

**不同类型成员的管理策略：**

| 成员类型 | 特征 | 发展策略 | 注意事项 |
|----------|------|----------|----------|
| 高潜型 | 技术强、有上进心、学得快 | 给最难的任务、加速晋升通道 | 注意不要"能者多劳"到过劳 |
| 稳健型 | 技术够用、准时交付、不追求晋升 | 尊重选择、在擅长领域深耕 | 不要强推他走管理路线 |
| 迷茫型 | 有潜力但不知道方向 | 多安排不同类型任务探索 | 用"试错"帮他找到热情点 |
| 停滞型 | 技术停滞、重复低水平工作 | 找到"最近发展区"给小挑战 | 警惕是否该转岗或淘汰 |
| 异动型 | 技术好但想去其他团队/公司 | 帮他分析利弊、做好知识交接 | 强扭的瓜不甜，体面分手 |

### ⚡ 实战经验

- **游戏行业的特殊性**：很多程序员入行是"想做游戏"的热情驱动。帮助成员保持这种热情比薪资激励更有效——让他参与玩法设计讨论、让他看到玩家反馈、让他为自己的代码上线后影响百万玩家而自豪
- **最常犯的错——"替"成员规划**：管理者画好蓝图然后让成员执行，这不是职业规划，这是派活。真正的职业规划是"提问→倾听→共创"，成员是主角，管理者是教练
- **"不想做管理"是合法选择**：不要给高级工程师施压"下一步该转管理了"。技术专家路线和管理路线同等重要，让想做技术的人安心做技术，是对人才最大的尊重
- **警惕 IDP 流于形式**：如果 IDP 写完就锁抽屉了，不如不写。关键不在文档，而在每两周的 1on1 里花 5 分钟问一句："上次说的那个学习目标，进展怎么样了？有什么我能帮的？"
