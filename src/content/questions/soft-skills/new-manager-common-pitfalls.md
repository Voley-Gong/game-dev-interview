## 参考答案

### ✅ 核心要点

1. **最大的坑：自己干活，不给团队机会** — 从"我来做"到"我来安排别人做"是最难的转变
2. **微观管理（Micromanagement）** — 关注每个实现细节，团队变成执行机器，丧失主动性
3. **回避困难对话** — 不敢给负面反馈、不敢处理摸鱼的人，问题越拖越大
4. **忽视向上管理** — 只顾埋头带团队，不主动和上级对齐期望，年终评估时才发现理解不一致
5. **技术停滞焦虑处理不当** — 要么完全放掉技术（失去判断力），要么抓着不放（变成瓶颈）

### 📖 深度展开

**五大经典陷阱详解：**

**陷阱一：超级英雄综合症**

> 表现：团队遇到难题，你说"放着我来"；review 代码时直接改掉而不是提建议；加班自己写业务逻辑。
>
> 本质原因：技术骨干的成就感来自"自己解决问题"，管理者的成就感应该来自"团队解决了问题"。
>
> 后果：团队成长慢 → 你越来越忙 → 成了瓶颈 → 团队依赖你 → 你更忙（死循环）

**转变路径：**

```
技术骨干心态：           技术管理者心态：
"这个问题我来解决"   →   "这个问题让谁来成长？"
"代码质量我来保证"   →   "我建立流程让质量被保证"
"我的产出 = 代码量"  →   "我的产出 = 团队总产出"
"我要是最强的人"     →   "我要让团队里出现比我强的人"
```

**陷阱二：不会给反馈**

> 典型场景：发现团队成员小王最近代码质量下降，你选择自己帮他改，而不是谈话。
> 三个月后小王离职，exit interview 说"我不知道自己哪里做得不好"。

**反馈的SBI模型（Situation-Behavior-Impact）：**

| 要素 | 说明 | 错误示范 | 正确示范 |
|------|------|----------|----------|
| S 情境 | 具体时间地点 | "你总是..." | "上周三的CR中..." |
| B 行为 | 客观描述行为 | "你代码写得很烂" | "有三个函数没有边界检查..." |
| I 影响 | 说明造成的影响 | "你太不负责了" | "导致测试环境崩溃，QA等了一下午" |

**陷阱三：回避冲突**

```
团队中的"毒瘤"员工类型：

类型              表现                          处理方式
─────────────────────────────────────────────────────────
消极抱怨型        每天传播负能量，影响士气        1on1了解根因，限期改善
推诿甩锅型        出了问题永远不是他的错          明确责任边界，记录在案
能力不匹配型      态度好但产出不达标              给予成长机会+时间盒考核
特权心态型        资历老、不做事但占资源          对事不对人，用数据说话
```

**关键原则**：一个管理者有没有威信，很大程度上看TA怎么处理"差员工"。团队都在看着——你处理不了刺头，优秀的人也会心寒。

**陷阱四：忽视向上管理**

> 场景：你带着团队拼命优化性能，把帧率从30提到了60。结果年底老板说"我更关心的是包体大小和崩溃率"。——目标没对齐。

**向上管理checklist：**
- ✅ 每月和上级做一次15分钟"期望校准"（你在意什么？什么是我做的好但没意识到的？）
- ✅ 每次大版本上线后主动汇报技术成果（附数据）
- ✅ 发现风险主动暴露（不要等到爆炸了再说）
- ✅ 了解上级的KPI是什么（你的工作如何支撑它？）

**陷阱五：技术判断力衰退**

> 管理者每天在开会，不碰代码。半年后技术评审时只能听别人说，无法判断方案优劣——你失去了技术管理者的核心竞争力。

**保持技术敏锐度的方法：**
- 每周固定半天"技术时间"（不排会，看代码、做技术调研）
- 坚持做CR（即使不写业务代码，至少review关键架构变更）
- 亲自参与技术POC和benchmark（不要全丢给团队）
- 关注行业技术动态（GDC演讲、技术博客、开源项目）

### ⚡ 实战经验

- **转型的前三个月最痛苦**：你会觉得"我今天什么也没做"——因为你的产出从代码变成了会议和沟通。这是正常的，给自己适应期
- **游戏行业的特殊性**：项目周期短、crunch期长，管理者更容易陷入"自己顶上"的陷阱，因为确实来不及培养人。但每次都这样，团队永远长不大
- **授权不是甩锅**：把任务交给团队时，说清"目标是什么、边界在哪、什么时候需要我介入"，而不是"这个你搞定，出了问题你负责"
- **找一个管理导师**：不管是公司里的前辈还是外部社区，管理这条路有人指路能少走很多弯路——很多坑前人都踩过
