## 参考答案

### ✅ 核心要点

1. **赛季生命周期管理**：每个赛季（Season）有明确的开始/结束时间戳，经历「预告期 → 活跃期 → 结算期 → 归档期」四个阶段。赛季切换时必须自动结算——未领取的奖励按策划配置决定补发或作废，上赛季的等级和领取记录持久化归档（冷存储），新赛季玩家从 0 级开始。赛季数据按赛季号分表，当前赛季热表、历史赛季冷归档。

2. **双轨奖励结构**：每个等级有两个奖励槽——免费轨（Free Track，所有玩家可领）和付费轨（Premium Track，购买战令后解锁）。购买时机灵活：可先攒等级再购买（补领之前等级的付费奖励），这种「先试后买」设计大幅提升转化率，但要求领取事务支持批量补发。

3. **经验引擎与等级曲线**：经验来源包括日常任务、周常任务、赛季挑战、对局结算，不同来源经验值差异巨大（日常 100~300、赛季挑战 2000~5000）。等级经验曲线常用线性（每级固定经验）或分段递增（前 50 级线性、后 50 级指数增长），等级上限通常 100 级。经验加成道具（双倍卡）需注意叠加规则（加法 vs 乘法）。

4. **任务驱动系统**：战令任务复用 Quest 系统的 Objective 模型，但任务池按赛季刷新（新赛季换一批任务）。任务分难度阶梯——每日任务（简单，如「参与 3 场对局」）、每周任务（中等）、赛季终极挑战（高难度，如「达成 50 次三杀」）。任务完成时触发经验发放事件，经验汇入战令等级进度。

5. **付费档位与购买事务**：常见三档——基础版（解锁付费轨）、豪华版（解锁 + 直升 25 级）、典藏版（解锁 + 直升 25 级 + 专属外观）。购买事务必须原子化（扣货币 → 标记解锁 → 补发已有等级的付费奖励 → 发放直升等级奖励），支持多端购买一致性（iOS/Android/PC 内购回调延迟差异大）。

6. **防脚本与经济平衡**：不设经验上限时，工作室脚本可 24 小时挂机刷满 100 级，破坏付费玩家的成就感。设置每日/每周经验上限（如每周封顶 20000 经验 ≈ 30 级），配合行为检测（同 IP 多账号、固定操作模式）封号。提供追赶机制——落后玩家经验加成（如落后 10 级以上时经验获取 +50%），保证活跃玩家都能接近满级。

### 📖 深度展开

#### 1. 赛季状态机与生命周期

```
赛季状态机：

  ┌───────────┐  到达开始时间  ┌──────────┐  到达结束时间  ┌────────────┐
  │ ANNOUNCE  │ ────────────→ │  ACTIVE  │ ────────────→ │ SETTLEMENT │
  │ (预告展示) │                │ (正常游玩) │                │ (锁定+补领) │
  └───────────┘                └──────────┘                └─────┬──────┘
                                                                 │ 结算完成
                                                                 ↓
                                                           ┌───────────┐
                                                           │ ARCHIVED  │
                                                           │(冷归档)    │
                                                           └───────────┘

  关键时间点：
  - 开始时间：新赛季解锁，玩家等级归零
  - 结束前1小时：锁定新购买（防最后一刻买豪华版白嫖直升）
  - 结束时间：停止获取经验
  - 结算期(24~72h)：玩家可补领上赛季未领奖励
  - 归档：结算期结束，数据移入冷存储
```

赛季切换结算流程：

```
触发结算：
  1. 扫描所有活跃玩家的上赛季数据
  2. 对每个玩家：
     ├── 有未领免费奖励 → 按配置补发或作废
     ├── 有未领付费奖励（且已购买）→ 强制补发到邮箱
     └── 写入赛季归档表（season_id, player_id, final_level, rewards_claimed）
  3. 重置玩家当前赛季等级 = 0
  4. 新赛季配置加载
  5. 广播新赛季开始事件
```

赛季数据分表策略：

| 数据 | 存储位置 | 访问模式 | 保留策略 |
|------|---------|---------|---------|
| 当前赛季进度 | 热表（Redis + DB） | 高频读写 | 赛季结束后迁移 |
| 当前赛季领取记录 | 热表（DB） | 中频读写 | 赛季结束后迁移 |
| 历史赛季归档 | 冷归档表（按月分区） | 极低频读 | 长期保留（客诉取证） |
| 赛季配置 | 配置表（静态） | 高频读 | 永久保留 |

#### 2. 双轨奖励与领取事务

```typescript
// 单个等级的奖励定义
interface BattlePassTier {
  seasonId: number;
  level: number;               // 等级 1~100
  expRequired: number;         // 升到本级所需累计经验
  freeReward: RewardItem[];    // 免费轨奖励
  premiumReward: RewardItem[]; // 付费轨奖励
}

// 玩家赛季进度
interface PlayerSeasonProgress {
  playerId: number;
  seasonId: number;
  currentLevel: number;
  currentExp: number;          // 当前等级内已获经验
  totalExp: number;            // 赛季累计经验（用于周上限判定）
  isPremium: boolean;          // 是否购买付费轨
  premiumTier: PremiumTier;    // Basic | Deluxe | Collector
  claimedLevels: number[];     // 已领取的等级列表（免费+付费合并记录）
  weekExp: number;             // 本周已获经验（防脚本）
  weekResetAt: number;         // 下周重置时间戳
}
```

领取奖励的事务化（核心防超发逻辑）：

```typescript
async function claimReward(playerId: number, level: number, track: 'free' | 'premium'): Promise<ClaimResult> {
  const progress = await getProgress(playerId);
  const tier = await getTierDef(progress.seasonId, level);

  // 1. 校验等级已达
  if (progress.currentLevel < level) return { ok: false, reason: 'level_not_reached' };

  // 2. 校验付费轨是否解锁
  if (track === 'premium' && !progress.isPremium) return { ok: false, reason: 'not_premium' };

  // 3. 校验是否已领取（幂等键防重复）
  const claimKey = `${progress.seasonId}:${level}:${track}`;
  if (progress.claimedLevels.includes(claimKey)) return { ok: false, reason: 'already_claimed' };

  // 4. 事务化发放：标记已领 + 发放奖励（原子操作）
  const reward = track === 'free' ? tier.freeReward : tier.premiumReward;
  await db.transaction(async (tx) => {
    await tx.addClaimRecord(playerId, claimKey);     // 标记已领
    await grantRewards(tx, playerId, reward);         // 发放（走背包/邮件）
  });

  return { ok: true, rewards: reward };
}
```

批量领取优化（玩家可能积攒 30+ 等级未领）：

```
串行方案（早期）：
  for level in unclaimedLevels:
    await claimReward(level)   # 逐级领取，每级一次事务
  → 30级 × 10ms/事务 = 300ms，玩家有明显卡顿

批量优化方案：
  1. 预校验：一次性查出所有可领等级（单次查询）
  2. 单事务批量发放：所有奖励合并到一个事务提交
  → 预校验 20ms + 批量发放 60ms = 80ms，无卡顿
```

#### 3. 经验引擎与防脚本

经验来源汇总：

  ┌─────────────┐   ┌─────────────┐   ┌──────────────┐   ┌──────────┐

         └────────┬────────┴──────────┬───────┘                │
                  ↓                   ↓                        ↓
            ┌─────────────────────────────────┐
            │      经验汇总 + 上限钳制         │
            │  (每日上限 + 每周上限 + 追赶加成) │
            └──────────────┬──────────────────┘

                    │ 等级提升计算   │
                    │ exp → level   │
                    └──────────────┘

等级经验曲线（三种常用模型）：

```typescript
// 线性曲线：每级固定经验，适合休闲游戏
function linearCurve(level: number, baseExp: number): number {
  return level * baseExp;  // 如 1000 * level，100级需10万经验
}

// 分段递增：前50级线性、后50级指数，前期快后期慢
function tieredCurve(level: number): number {
  if (level <= 50) return level * 1000;           // 线性段
  const excess = level - 50;
  return 50000 + Math.floor(1000 * Math.pow(1.05, excess)); // 指数段
}

// 追赶加成：落后玩家经验倍率
function catchUpBoost(playerLevel: number, avgLevel: number): number {
  const gap = avgLevel - playerLevel;
  if (gap <= 10) return 1.0;         // 差距小，无加成
  if (gap <= 30) return 1.5;         // 中等落后，+50%
  return 2.0;                        // 严重落后，+100%
}
```

防脚本经验上限配置：

| 限制维度 | 上限值 | 目的 | 效果 |
|---------|-------|------|------|
| 每日经验上限 | 5000（≈5 级） | 防单日爆肝 | 脚本无法日刷 24 级 |
| 每周经验上限 | 20000（≈30 级） | 防周刷满 | 100 级需 3~4 周 |
| 对局经验衰减 | 连续 20 局后经验 ×0.5 | 防挂机刷局 | 挂机收益递减 |
| 行为检测 | 同 IP >3 账号标记 | 识别工作室 | 配合封号 |

### ⚡ 实战经验

1. **赛季结算补领窗口**：赛季结束前 1 小时锁定新购买（防止玩家在最后时刻买豪华版白嫖直升 25 级的奖励），结算后设 24~72 小时补领期。曾因补领期仅 2 小时，大量上班族错过，赛季相关客诉中「未领奖励过期」类占 35%；延长到 48 小时后降至 5% 以下。

2. **批量领取性能优化**：活跃玩家可能攒一周不领，积攒 30+ 等级后一次性领取。早期串行方案（逐级领取，每级独立事务）耗时 300ms，玩家有明显卡顿感。优化为预校验 + 单事务批量发放后降到 80ms，体验提升显著。

3. **经验上限防工作室**：未设上限时，工作室脚本 24 小时挂机，3 天刷满 100 级并出售满级账号，严重破坏付费玩家（花 100 元买豪华版）的成就感。设置每周经验上限（20000 经验 ≈ 30 级/周）+ 对局经验连续衰减后，工作室刷满周期从 3 天延长到 28 天以上，账号交易价从 50 元跌到 5 元，工作室基本放弃。

4. **多端购买一致性**：iOS 内购回调延迟可达 30 秒（苹果服务器排队），玩家在 iOS 购买后立即切到 PC 端，可能看不到付费轨已解锁，导致重复购买。解决方案：服务端解锁状态主动推送 + 客户端购买后轮询（每 3 秒查一次解锁状态，超时 15 秒提示「购买处理中，请稍后」）。上线后重复购买率从 2% 降到 0.1%。

5. **直升等级的奖励补发**：玩家购买豪华版（直升 25 级）时，1~25 级的付费奖励必须全部补发。早期实现遗漏了「补发」逻辑，只解锁了付费轨但没发 1~25 级奖励，导致大量客诉。补发逻辑需复用批量领取事务，确保幂等（已领的不重复发）。
