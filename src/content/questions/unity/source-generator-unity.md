## 参考答案

### ✅ 核心要点

1. **Source Generator** 是 Roslyn 编译器插件，在编译期分析用户代码并生成新的 C# 源文件，零运行时反射开销
2. Unity 2022.2+ 正式支持 Source Generator（需要 .NET Standard 2.1 / C# 9+ 配置）
3. 典型应用：自动生成 `SerializeReference` 类型标签、DI 容器注册、状态机代码、消息路由、DTO Mapper
4. 生成的代码参与编译，有完整的 IntelliSense 和类型检查，比 T4 模板和 Reflection.Emit 更安全
5. 与 ILPostProcessor（基于 IL 后处理）互补：Source Generator 偏"增量、增量友好"，ILPostProcessor 可做更深层的 IL 织入（如 ILRuntime、Fody）

### 📖 深度展开

#### Unity 中启用 Source Generator

```xml
<!-- 在 .asmdef 同级放置 .asmdef 引用 Generator 项目 -->
<!-- Generator 项目结构 -->
<!--
  MyGenerator/
    MyGenerator.asmdef  (Unity.ScriptableCompilerSupport 包)
    MyGenerator.csproj
    Generators/
      MyGenerator.cs
-->

<!-- Generator csproj 配置 -->
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>netstandard2.0</TargetFramework>
    <!-- 必须引用 Roslyn Analyzer API -->
    <IsRoslynComponent>true</IsRoslynComponent>
    <LangVersion>latest</LangVersion>
  </PropertyGroup>
  <ItemGroup>
    <PackageReference Include="Microsoft.CodeAnalysis.CSharp" Version="4.3.0" />
  </ItemGroup>
</Project>
```

#### 一个实际的 Generator：自动生成类型注册

```csharp
// —— Generator 端 ——
[Generator]
public class AutoRegisterGenerator : IIncrementalGenerator
{
    public void Initialize(IncrementalGeneratorInitializationContext context)
    {
        // 扫描带 [AutoRegister] 特性的类
        var classProvider = context.SyntaxProvider
            .ForAttributeWithMetadataName(
                "AutoRegisterAttribute",
                predicate: (node, _) => node is ClassDeclarationSyntax,
                transform: (ctx, _) => (INamedTypeSymbol)ctx.TargetSymbol);

        context.RegisterSourceOutput(classProvider.Collect(), (spc, symbols) =>
        {
            var sb = new StringBuilder();
            sb.AppendLine("// <auto-generated/>");
            sb.AppendLine("namespace AutoRegistration");
            sb.AppendLine("{");
            sb.AppendLine("    public static class Registry");
            sb.AppendLine("    {");
            sb.AppendLine("        public static void RegisterAll(IContainer container)");
            sb.AppendLine("        {");
            foreach (var sym in symbols)
            {
                sb.AppendLine($"            container.Register<{sym.ToDisplayString()}>();");
            }
            sb.AppendLine("        }");
            sb.AppendLine("    }");
            sb.AppendLine("}");

            spc.AddSource("AutoRegistry.g.cs", sb.ToString());
        });
    }
}
```

```csharp
// —— 业务代码端 ——
[AutoRegister]
public class EnemySpawner : ISpawner { /* ... */ }

[AutoRegister]
public class LootSystem : ISystem { /* ... */ }

// 编译后自动存在 Registry.RegisterAll() 方法，零反射
```

#### Source Generator vs 其他代码生成方案

| 方案 | 时机 | 优点 | 缺点 |
|------|------|------|------|
| **Source Generator** | 编译期（C# 源码级） | 增量编译友好、类型安全、无反射 | 只能添加代码，不能修改已有代码 |
| **T4 模板** | 设计时 / 构建前 | 简单直观、模板灵活 | 不参与增量编译、维护成本高 |
| **ILPostProcessor** | 编译后（IL 级） | 可修改任意 IL | 调试困难、性能开销大 |
| **Reflection.Emit** | 运行时 | 最大灵活性 | 无编译期检查、启动慢、AOT 不兼容 |
| **Burst Compile** | 编译后（Job/Span） | SIMD + 排除 GC | 仅限 DOTS 生态 |

#### Unity 项目中的常见应用模式

```
模式1: 消息/事件路由
  [HandleMessage] → 生成 MessageDispatcher.Switch(msgType)

模式2: SerializeReference 类型标签
  [SerializeReference] 子类 → 生成 SubclassPicker 下拉菜单代码

模式3: 表驱动配置
  Excel/JSON → Source Generator → 编译期生成的 readonly struct

模式4: 状态机
  [State] + [Transition] → 生成 StateMachine.Switch/Update 方法
```

### ⚡ 实战经验

- **用 IIncrementalGenerator 而非 ISourceGenerator**：增量生成器（C# 10+）支持增量编译缓存，改一行代码不会重新生成全部文件
- **Generator DLL 不能直接放 Unity 项目**：需要单独的 .csproj（Roslyn 组件），通过 `.asmdef` 配合 `Unity.ScriptableCompilerSupport` 包接入
- **生成的代码无法断点调试**：出问题时用 `// <auto-generated/>` 标记 + 重写 `.WithTrackingName()` 输出诊断信息
- **IL2CPP 下避免在 Generator 中使用 `dynamic`**：生成的代码最终走 IL2CPP 编译，所有类型必须在编译期确定
- **调试技巧**：在 csproj 中设置 `<EmitCompilerGeneratedFiles>true</EmitCompilerGeneratedFiles>`，可查看生成的 .g.cs 文件
